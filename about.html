About The Portal

The web site is designed with privacy and security at the foremost, even before functionality. After all, many of the functions of our site can be achieved using a combination of other web services like our YRDSB GAPPS account, Discord group etc. If members have to give up more personal info or get exposed to additional security risks then we are better off not having the site.

User-Ids and Passwords

Our site collects or stores ZERO personal information on the server. Not even your user-Id, and most definitely not passwords. User-Ids or passwords are not even communicated across the network during authentication. Therefore there is no chance for hackers to eavedrop on anything or steal from our server, because there is nothing to capture or steal! You may wonder if it would be better to do away with signing-in altogether. In order to protect against abuse and for fairness in certain functions like polling, we do sometimes require members to authenticate their membership through signing-in, and to keep track of polls and messages, but in an anonymous way, as far as possible. We only send irreversibly-hashed user-Ids and twice irreversibly-hashed passwords to the server for authentication. On the server side, we followed the best practice of storing and comparing only further "salted" and industrial-strength Blowfish-hashed values in our user-list. The key idea is that plain user-Ids or passwords are NEVER even sent through the network, and that the hashed passwords are further salted (i.e. with long random bits added) on the server side to eliminate the possibility of even telling whether two password hashes came from the same original password.
The site is intended to provide a unified location for Math Club resources like lesson notes, problems and solutions, list of events, announcements etc. These used to be placed in some YRDSB directories and they require signing onto a school account for access.

Our portal site would hopefully provide such access more simply and more. For example, we could set up simple polls for not so serious issues and allow members to vote anonymously or authenticated.
Interactive activities can also be done. E.g. games, animated demos, quizzes etc.

Member contributed contents

The site is only as useful as members feel it to be. Therefore it is important to allow as much member participation and contribution as possible.

The Event Calendar is already fully editable by any authenticated users. We are thinking that the News interface would be opened for editing by any members and that new Quiz questions can be contributed to be automatically listed under the "Pick Some Topics" interface. A Math Micro-blog is being considered too. More about that later!

The Event Calendar interface is now fully functional! Please try it. You could even use asciiMath (or the more tedius LaTeX) to put in tricky math expressions...

Sign-in and you could contribute any note, reminder or contest info by adding new events into the calender.
General members would be able to modify only their own contributed events. If you are interested to help with the maintainance of the portal data, including moderating and updating events, please contact Rufin to upgrade your account to admin level — i.e. `aleph_0`!

The News list is for displaying the latest news and announcements, or even the latest weekly problems.

It is now fully editable through the Docs  interface (after you have signed in as a user with admin rights). Just click on the News tab  on top of the Doc List dialog, and there you can create new News Articles, choose an auto-expiration date, and pick your bespokeon icon for your piece of announcement — with MathJax (AsciiMath + LaTeX) typesetting support too!

Please let Rufin know if there are problems with the new editor. (Although it has been tested on tiny screen smartphones, it is probably not very practical to edit complex news articles on a cramped phone screen — a few lines of quick announcements should be fine.)

Polling is working! Please try it.
Setting up a new poll is really easy. Just add a pair of files under the "docs/polls" directory:
  1. A Poll File, for defining the active-until time, filename of a poll description file and a sequence of poll choices;
  2. A corresponding description file (text with formatting markups) referenced from the Poll File.
The Quiz page automatically reads in question files in the form of an array of JS objects. The format is quite simple. Please check out the quiz file: MCQ_Misc.js for some examples.
Anyone is welcome to contribute new questions under any categories -- as long as the questions are well tested, correct and interesting!

The Portal is implemented as a Progressive Web App (PWA) that can be installed on the home page of mobile devices. The client (browser) side is written in TypeScript (compiled into JavaScript then further "advanced closure-compiled" into tightly optimized JS codes using Google's Closure Compiler).

The server side is written in Php. We are currently not sure if the hosting service provider (IONOS.ca) allows Server Sent Events (SSE). If so, we could write a chat room interface that allows Math equations to be typed out as 1st class citizens!
